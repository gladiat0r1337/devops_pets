 Автоматизированный деплой статического сайта

Этот проект демонстрирует полный цикл базовой настройки веб-сервера и автоматизации доставки кода. В качестве веб-сервера используется Nginx, а для синхронизации файлов между локальным компьютером и удаленным сервером — утилита rsync поверх безопасного SSH-соединения по ключам.

Локальная структура проекта

site/
├── src/                 # Папка с исходниками сайта
│   ├── index.html        # Главная HTML-страница
│   └── style.css         # Файл стилей
└── deploy.sh             # Bash-скрипт для автоматического деплоя


Шаг 1. Локальная настройка SSH

Чтобы деплой работал в одну кнопку, нужно настроить беспарольный доступ по ключу и создать алиас для сервера.

Генерация ключа: ssh-keygen -C "" -f

Публичный ключ необходимо перенести на сервер по пути: ~/.ssh/authorized_keys.

Ключ должен быть защищен(ssh может ругаться и не пропускать по данному ключу, если задетекти пролемы с безопасностью ключа):

chmod 600 ~/.ssh/твой_ключ

Для удобства подключения на локальной машине можно добавить конфиг файл: ~/.ssh/config:

Host host_name
    HostName <IP_СЕРВЕРА>
    User root
    IdentityFile ~/.ssh/<имя_приватного_ключа>

После выполнения данного шага можно будет подключаться к серверу через команду ssh host_name

Шаг 2. Настройка удаленного сервера (VPS)

1. Установка Nginx

sudo apt update
sudo apt install nginx -y


2. Подготовка директории для сайта

Создание папки и выдача прав (Nginx должен читать файлы, а юзер — записывать):

mkdir -p /var/www/mysite
chown -R root:www-data /var/www/mysite
chmod -R 750 /var/www/mysite


3. Настройка конфигурации Nginx

nano /etc/nginx/sites-available/mysite

Файл должен содержат слеующий код:

server {
    listen 80; - прослушка порта для IPv4
    listen [::]:80; - аналогично для IPv6

    root /var/www/mysite; - папка с файлами сайта
    index index.html; - основная страница которую должен отображать Nginx

    server_name _; - _, так как у сайта нет домена(отвечать на любой домен/IP)

    location / {
        try_files $uri $uri/ =404;
    } - обработка запросов через try_files позволяет анализировать запрос юзера и в случае отсутсвия указанного в запросе файла или папкп отдавать ошибку
}


4. Активация сайта

Создание мягкой ссылки на конфиг сайта(Nginx ищет файлы именно из папки sites-enabled)
Удаление дефолтного приветсвия Nginx

ln -s /etc/nginx/sites-available/mysite /etc/nginx/sites-enabled/
rm /etc/nginx/sites-enabled/default

Проверка и перезапуск Nginx:

nginx -t
systemctl reload nginx


Шаг 3. Деплой сайта

Скрипт deploy.sh позволяет одной командой отправить все изменения из папки src/ на удаленный сервер.

deploy.sh:

#!/bin/bash

echo "Deploy data on server"

rsync -az --delete ./src root@vds_conn:/var/www/mysite

if [ $? -eq 0 ]; then
	echo "Deployed succsessfuly"
else
	echo "Error ocured"
fi


Для запуска необходимо дать права на исполнения файла:

chmod 755 deploy.sh

Запуск:

./deploy.sh
